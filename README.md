## üéØ Overview

**n8n MCP Server** is a production-ready Model Context Protocol server that bridges Claude AI with n8n workflow automation. It enables natural language workflow creation, intelligent template search, and complete workflow lifecycle management.

### What is MCP?

The [Model Context Protocol](https://modelcontextprotocol.io/) (MCP) is an open standard by Anthropic that enables AI assistants to securely interact with external tools and data sources.

### Why This Project?

- **üé® Natural Language Workflows**: Create complex automations by simply describing what you want
- **üîç Smart Discovery**: Search through 541 n8n nodes and 2700+ templates instantly
- **‚ö° Production Ready**: Type-safe, tested, and containerized for easy deployment
- **üß† AI-Powered**: Automatically matches your requirements with the best nodes and templates
- **üê≥ Docker Native**: One-command deployment with full isolation

## üöÄ Quick Start

### Option 1: Docker (Recommended)

The fastest way to get started:

```bash
# 1. Clone the repository
git clone https://github.com/aurelienfagioli/n8n-mcp-aurelien.git
cd n8n-mcp-aurelien

# 2. Configure your n8n credentials
cp .env.example .env

# 3. Initialize database
npm install
npm run db:init

# 4. Build Docker image
npm run docker:build

# 5. Configure Claude Desktop
```

## üì¶ Installation

Choose your preferred installation method:

```bash

npm run db:init
```


```bash
docker run -i --rm \
```

## ‚öôÔ∏è Configuration

### Configure Claude Desktop

Add the MCP server to Claude Desktop's configuration file:

**File Locations:**
- **macOS**: `~/Library/Application Support/Claude/claude_desktop_config.json`
- **Windows**: `%APPDATA%\Claude\claude_desktop_config.json`
- **Linux**: `~/.config/Claude/claude_desktop_config.json`

**For Docker Deployment:**

```json
{
  "mcpServers": {
    "n8n": {
      "command": "docker",
      "args": [
        "run", "-i", "--rm",
        "-e", "N8N_API_URL=https://your-n8n-instance.com",
        "-e", "N8N_API_KEY=your_api_key_here",
        "-v", "n8n-mcp-data:/app/data",
        "ghcr.io/aurelienfagioli/n8n-mcp-aurelien:latest"
      ]
    }
  }
}
```

**For Local Node.js Installation:**

```json
{
  "mcpServers": {
    "n8n": {
      "command": "node",
      "args": [
        "/absolute/path/to/n8n-mcp-aurelien/dist/index.js"
      ],
      "env": {
        "N8N_API_URL": "https://your-n8n-instance.com",
        "N8N_API_KEY": "your_api_key_here",
        "DATABASE_PATH": "/absolute/path/to/n8n-mcp-aurelien/data/nodes.db"
      }
    }
  }
}
```

‚ö†Ô∏è **Important**: Replace `your-n8n-instance.com` and `your_api_key_here` with your actual credentials!

**Restart Claude Desktop** after saving the configuration.

## üêõ Troubleshooting

### Common Issues

#### ‚ùå "Failed to connect to n8n API"

**Symptoms:** MCP server can't reach n8n instance

**Solutions:**
1. Verify `N8N_API_URL` is correct and accessible
2. Check API key is valid: Settings ‚Üí API ‚Üí API Keys
3. Test connection manually:
   ```bash
   curl -H "X-N8N-API-KEY: your_key" \
        https://your-n8n.com/api/v1/workflows
   ```
4. Ensure n8n API is enabled in settings

#### ‚ùå "Database file not found"

**Symptoms:** `ENOENT: no such file or directory`

**Solution:**
```bash
npm run db:init
```

#### ‚ùå "Claude Desktop doesn't see the server"

**Symptoms:** MCP tools not appearing in Claude

**Solutions:**
1. Check config file path is correct for your OS
2. Verify JSON syntax is valid (use JSONLint)
3. Use **absolute paths** (not `~` or `./`)
4. **Completely quit** Claude Desktop (not just close window)
5. Check logs: Claude Desktop ‚Üí Settings ‚Üí Developer

#### ‚ùå "Docker container exits immediately"

**Symptoms:** Container starts and stops

**Solutions:**
```bash
# Check logs
docker logs n8n-mcp-aurelien

# Verify environment variables
docker inspect n8n-mcp-aurelien

# Ensure database exists in volume
docker volume inspect n8n-mcp-data
```

#### ‚ùå "Permission denied" on Docker

**Symptoms:** Can't run docker commands

**Solutions:**
```bash
# Linux/macOS: Add user to docker group
sudo usermod -aG docker $USER
# Log out and back in

# Or run with sudo (not recommended)
sudo docker run ...
```

### Debug Mode

Enable verbose logging:

```bash
# In .env file
LOG_LEVEL=debug

# Or via environment variable
LOG_LEVEL=debug npm start
```

## üîí Security

### Best Practices

- ‚úÖ **Never commit** `.env` files or API keys to Git
- ‚úÖ Use **environment variables** for all secrets
- ‚úÖ Run containers as **non-root user** (already configured)
- ‚úÖ Regularly **update dependencies**: `npm audit fix`
- ‚úÖ Use **HTTPS** for n8n API endpoints
- ‚úÖ Rotate API keys periodically

### Docker Security

The Dockerfile includes security best practices:

```dockerfile
# Non-root user
RUN adduser -S n8nmcp && chown -R n8nmcp /app
USER n8nmcp

# Minimal base image
FROM node:20-alpine

# No unnecessary privileges
# Read-only filesystem where possible
```

### Data Privacy

- üîê API keys stored in environment variables only
- üîê SQLite database contains only **public** n8n node/template data
- üîê No workflow data or user credentials stored locally
- üîê All API calls use secure HTTPS connections
